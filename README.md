## Что это

MorseWave — веб-тренажёр для изучения азбуки Морзе с нуля до приёма на слух
реальных позывных. Без фреймворков на фронтенде и без Bootstrap: чистый JS
(Web Audio API) + собственная тёмная дизайн-система.

Прогресс (XP, уровень, серия дней, выученные символы) хранится локально
в браузере — регистрация не нужна. MySQL используется под капотом для
справочных данных (достижения, банк позывных) и анонимной статистики
сообщества.

## Возможности

| Раздел | Что внутри |
|---|---|
| 📖 **Буквы** | Учи каждый символ отдельно: отправка ключом (тап/пробел, с живым сайдтоном) и приём на слух (играет символ → нажми на экранной клавиатуре) |
| 🎯 **Метод Коха** | Символы сразу на целевой скорости; новый символ открывается при точности ≥90%; можно вручную установить любое число открытых символов (в обе стороны) |
| 🔢 **Группы символов** | Приём на слух групп по 2–5 символов: буквы / цифры / микс / только выученные, скорость и интервалы Фарнсворта — слайдерами |
| 📡 **Позывные** | Тренировка на реальных по формату позывных (включая российские форматы), банк расширяется скриптом-генератором |
| 🏆 **Достижения** | 17 ачивок, условия хранятся в БД — можно добавлять новые без правки кода |
| 🎯 **Задание дня** | Единый набор параметров тренировки на сегодня для всех + бонус XP |
| 💡 **Сигнальная лампа** | Визуальный аналог звука — можно тренироваться совсем без звука |
| ⚠️ **Сброс прогресса** | Кнопка на странице «Достижения» — полный сброс localStorage с подтверждением |

## Стек

- **PHP 8+** — без фреймворка, простой роутинг по файлам
- **MySQL / MariaDB** — через PDO
- **Vanilla JS** — Web Audio API для звука, без сборки и npm
- **Кастомный CSS** — дизайн-система в `assets/css/style.css`, без Bootstrap
- **Docker** — для локальной разработки (PHP+Apache, MySQL, phpMyAdmin)

## Быстрый старт (Docker)

Нужен только [Docker Desktop](https://www.docker.com/products/docker-desktop/) —
никакого Laragon/OpenServer и ручной настройки MySQL.

```bash
git clone <URL_ТВОЕГО_РЕПО> morse-trainer
cd morse-trainer
docker compose up -d --build
```

Готово:
- Сайт — **http://localhost:8080**
- phpMyAdmin — **http://localhost:8081** (сервер `db`, логин `root`,
  пароль `morsewave_root`)

При первом запуске (на пустой БД) MySQL-контейнер сам накатывает
`database/schema.sql` и все `database/migration_*.sql` по порядку — руками
ничего заливать не нужно. Это происходит один раз; при повторных
`docker compose up` существующие данные не трогаются.

`config/database.php` для Docker создавать не нужно — `docker-compose.yml`
прокидывает креды через переменные окружения (`DB_HOST=db` и т.д.), которые
`config/database.php` подхватывает автоматически (см. `config/database.example.php`).
Если запускаешь без Docker — см. ниже.

**Почта (для писем-подтверждений аккаунта)** — опционально для локальной
разработки, обязательно для прода:
```bash
cp config/mail.example.php config/mail.php
```
и впиши реальный API-ключ [Resend](https://resend.com/api-keys) (бесплатный
тариф — сотен писем в месяц достаточно). Без домена по умолчанию письма
уходят с sandbox-адреса `onboarding@resend.dev` — работает сразу, без
дополнительной настройки DNS. Без этого шага аккаунты по-прежнему создаются,
письма просто не отправляются — ссылка на подтверждение падает в
`error_log` (`docker compose logs -f app`), можно подтвердить вручную при
разработке.

Опционально — сгенерировать побольше позывных:
```bash
docker compose exec app php database/seed_callsigns.php 200
```

Остановить: `docker compose down` (данные останутся в volume).
Стереть всё и начать с нуля: `docker compose down -v`.

## Шпаргалка по Docker-командам

Всё, что реально нужно в повседневной работе с этим проектом — одним списком.

**Запуск / остановка**
```bash
docker compose up -d --build   # собрать и запустить (после изменений в Dockerfile/зависимостях)
docker compose up -d           # запустить без пересборки (обычный случай)
docker compose down            # остановить (данные в volume остаются)
docker compose down -v         # остановить И стереть БД — начать с чистого листа
docker compose restart app     # перезапустить только контейнер с сайтом (после правки конфигов Apache)
```

**Статус и логи**
```bash
docker compose ps              # какие контейнеры сейчас подняты
docker compose logs -f app     # логи PHP/Apache в реальном времени (Ctrl+C — выйти)
docker compose logs -f db      # логи MySQL
docker compose logs --tail=50 app   # последние 50 строк, без "хвоста"
```

**Заглянуть внутрь / выполнить команду в контейнере**
```bash
docker compose exec app bash                          # зайти в контейнер сайта командной строкой
docker compose exec app php -l learn.php               # проверить синтаксис конкретного PHP-файла
docker compose exec app php database/seed_callsigns.php 200   # сгенерировать позывные
docker compose exec db mysql -uroot -pmorsewave_root morse_trainer   # зайти в MySQL консольно
```

**Накатить миграцию вручную** (обычно не нужно — новые миграции подхватываются
автоматически только на ПУСТОЙ базе; если база уже существует, новую
миграцию нужно применить руками одним из способов):
```bash
# через phpMyAdmin (проще): http://localhost:8081 → вкладка SQL → вставить содержимое файла

# либо через консоль:
docker compose exec -T db mysql -uroot -pmorsewave_root morse_trainer < database/migration_7.sql
```

**Если что-то совсем сломалось — полный сброс**
```bash
docker compose down -v
docker compose up -d --build
```
(это пересоздаст контейнеры и базу с нуля — код из volume-примонтированной
папки проекта, естественно, никуда не денется, сотрутся только данные MySQL)

## Как это устроено внутри

- **`assets/js/audio.js`** — `MorseAudio` проигрывает точки/тире через
  осциллятор Web Audio API, поддерживает интервалы Фарнсворта (символы на
  одной скорости, паузы — на другой, более медленной). `onSymbol` колбэк
  отдаёт `durationMs` — используется для точной синхронизации визуализации.
- **`assets/js/input.js`** — `TelegraphKey` превращает удержание клавиши
  (тап по экрану или пробел) в точки/тире по длительности нажатия
  относительно текущего `wpm`, плюс живой сайдтон (реальный звук) во время
  удержания. Пороги: `< 2×unit` → точка, иначе тире; пауза `2.5×unit` —
  граница буквы, ещё `+4×unit` — граница слова.
- **`assets/js/signal.js`** — `SignalLine` (осциллограф из точек/тире) и
  `MorseLamp` (визуальная лампа) — сквозные визуальные элементы, синхронно
  отражающие и звук, и ввод на всех страницах тренировки.
- **`assets/js/progress.js`** — весь прогресс пользователя в одном ключе
  `localStorage` (`morsewave_progress_v1`): XP, уровень, серия дней,
  выученные символы, уровень Коха, статистика, разблокированные
  достижения. `Progress.resetAll()` — полный сброс.
- **Достижения** определяются в MySQL (`achievements`, поле `condition_type`
  + `condition_value`), а проверка условий против статистики происходит на
  клиенте в `Progress.checkAchievements()` — сопоставление типов см. в
  `progress.js` (`statValue()`).
- **XP анти-фарм**: в `learn.js` XP за отправку символа начисляется только
  пока символ ещё не в `learnedLetters`; после освоения повторы бесплатны.

## Деплой на VPS

```bash
cd /var/www
git clone <URL_ТВОЕГО_РЕПО> morse-trainer
cd morse-trainer
cp config/database.example.php config/database.php
cp config/mail.example.php config/mail.php
# впиши реальный API-ключ Resend в config/mail.php — иначе письма
# подтверждения не будут отправляться (ссылка упадёт в error_log)
mysql -u root -p < database/schema.sql
mysql -u root -p morse_trainer < database/migration_2.sql
mysql -u root -p morse_trainer < database/migration_3.sql
mysql -u root -p morse_trainer < database/migration_4.sql
mysql -u root -p morse_trainer < database/migration_5.sql
mysql -u root -p morse_trainer < database/migration_6.sql
mysql -u root -p morse_trainer < database/migration_7.sql
mysql -u root -p morse_trainer < database/migration_8.sql
mysql -u root -p morse_trainer < database/migration_9.sql
```

Дальше:

1. Виртуальный хост Apache/Nginx на папку проекта (document root — корень
   репозитория, где лежит `index.php`).
2. Убедиться, что установлены `php`, `pdo_mysql`.
3. Обновить `config/database.php` под продакшн-креды MySQL (для публичного
   сервера лучше вынести их в переменные окружения).
4. Обновления — `git pull` на сервере после `git push` с локальной машины,
   плюс накатить новые `migration_*.sql`, если они появились.
5. **Apache**: в конфиге виртуального хоста должно быть `AllowOverride All`
   (или хотя бы `AllowOverride AuthConfig Limit`) для директорий `config/`
   и `database/` — иначе `.htaccess`-запреты на эти папки не сработают.
   Большинство панелей управления (FastPanel, ISPmanager, cPanel) включают
   это по умолчанию для пользовательских сайтов.
   **Nginx** `.htaccess` не читает вообще — если сервер на Nginx, запрет
   нужно продублировать в конфиге сайта:
   ```nginx
   location ~ ^/(config|database)/ { deny all; return 404; }
   ```

## История изменений

- **v1 — первая версия.** Буквы (отправка ключом), метод Коха, группы
  символов 2–5, позывные, XP/уровни/серия дней, 15 ачивок. Прогресс в
  localStorage, справочные данные — в MySQL.
- **v2.**
  - Новый режим «Приём на слух» на странице «Буквы» + 2 ачивки
    (`migration_2.sql`).
  - Сайдтон при отправке ключом; звук символа при верном повторе.
  - Порог «выучено» 3 → 5 повторов; фикс фарма XP на выученных символах.
  - Визуальная лампа (`MorseLamp`) во всех тренажёрах; сигнальная линия и
    лампа сбрасываются перед каждым проигрыванием.
  - Скорость — слайдеры с шагом 1 везде; Фарнсворт — чекбокс + слайдер.
  - Ввод ответа в Кохе/группах/позывных доступен сразу, не дожидаясь конца
    воспроизведения.
  - Ручная установка уровня Коха; российские форматы позывных +
    генератор `seed_callsigns.php`; «Задание дня» на главной.
  - README/LICENSE под GitHub, `config/database.php` вынесен из git.
- **v2.1 — багфиксы (`migration_3.sql`).**
  - Ачивка «Мастер Коха» требовала 40 символов при 38 существующих —
    была недостижима. Исправлено на 38.
  - Слайдер ручной установки уровня Коха разрешал двигать только вперёд —
    теперь работает в обе стороны (с подтверждением при уменьшении).
  - Добавлена кнопка «Сбросить весь прогресс» на странице «Достижения».
- **v2.2 (`migration_4.sql`).**
  - «Приём на слух»: фикс дефолтного набора символов (был жёсткий фолбэк
    K/M вместо всего алфавита) + переключатель набора (все / буквы / цифры
    / только выученные).
  - Ввод ответа с физической клавиатуры в «Приёме на слух» — не только тап
    по плитке.
  - Мини-дашборд статистики сессии: текущая серия, рекорд серии (хранится
    вечно, `stats.recognizeBestStreak`), точность за сессию, всего верно.
  - 2 новые ачивки за рекорд серии: «Начинающий телеграфист» (50 подряд) и
    «Телеграфист со стажем» (500 подряд).
- **v2.3 — багфиксы, без изменений схемы БД.**
  - **Баг статистики на главной**: XP/уровень/серия дублировались в шапке
    и на главной, `renderNavStats()` обновлял только первый найденный
    элемент (`querySelector`) — второй оставался с дефолтным «0». Исправлено
    на `querySelectorAll`.
  - **Баг статистики позывных**: XP и счётчики (`callsignsCompleted`,
    `total_callsigns`) начислялись только по завершении всей сессии — если
    сессия не пройдена до конца, верные ответы не засчитывались. То же
    исправлено в группах и методе Коха: теперь начисляется сразу за каждый
    верный ответ, а не пачкой в конце (кроме бонуса за точность сессии
    в Кохе и бонуса задания дня — они по-прежнему требуют полного
    прохождения).
  - Защита от спама кнопкой «Повторить»/«Прослушать» — пока предыдущий
    сигнал не доиграл, повторный клик игнорируется (кнопка также блокируется
    визуально).
  - «Приём на слух»: кнопка «Играть новый символ» (которую можно было
    жать, чтобы пропустить символ без ответа) заменена на «Начать
    тренировку» / «Остановить» — непрерывный поток символов, ответил →
    сразу следующий.
  - «Отправка ключом»: под паттерном добавлены строки «Ти-Та» и русский
    «напев» (например, A — «ай-да», B — «бакитекут»), подсвечиваются
    слог в слог в такт проигрыванию.
- **v2.4 — баланс XP и серия дней.**
  - **XP-фарм в «Буквах» (отправка ключом)**: раньше опыт начислялся за
    каждую верную попытку, поэтому можно было специально ошибаться на
    последнем повторе из пяти и фармить бесконечно. Теперь XP (+25)
    начисляется только один раз — за реально пройденную серию из 5 подряд
    без единой ошибки.
  - **Серия дней («стрик»)** раньше засчитывалась от любого действия
    (даже одного тапа в «Приёме на слух»). Теперь день засчитывается
    только за реальную тренировку: завершённую сессию Коха/групп/позывных
    или изучение новой буквы (`Progress.markDailyActivity()`); простое

## Известные ограничения / открытые вопросы

- Аккаунты — опциональная надстройка над localStorage (не обязательны для
  тренировки). Один и тот же прогресс из одного браузера технически можно
  опубликовать от нескольких аккаунтов (дубли в лидерборде) — жёсткой
  защиты нет, только email-подтверждение как барьер против моментального
  создания кучи аккаунтов + мягкое предупреждение (`confirm()`) при
  повторной публикации с тем же браузером под другим аккаунтом.
- Письма подтверждения шлются через **Resend API** (`config/mail.php`,
  `includes/resend_mailer.php`) — надёжнее и SMTP через shared-хостинг
  (там был конкретный кейс: у `mail.r9old.ru` не настроен DKIM, а PTR
  общего IP вообще не указывал на домен — Gmail тихо резал письма).
  Без верификации домена в Resend письма уходят с sandbox-адреса
  `onboarding@resend.dev` — работает сразу, но выглядит не как «родной»
  адрес; для `noreply@r9old.ru` нужно верифицировать домен в Resend
  (Domains → Add Domain, добавить их DNS-записи). В Docker локально
  `config/mail.php` по умолчанию не настроен (ключ-плейсхолдер) — писем
  не будет, ссылка на подтверждение падает в `error_log`
  (`docker compose logs -f app`). Восстановление забытого пароля по почте
  (отдельный от подтверждения e-mail функционал) пока не реализовано.
  Старый SMTP-клиент (`includes/smtp_mailer.php`) удалён как избыточный —
  если понадобится SMTP-путь снова, см. историю в changelog v2.20/v2.21.
- Капча на азбуке Морзе (`includes/captcha.php`) — базовая защита от
  примитивных скриптов, не от целенаправленного обхода. Этого достаточно
  для небольшого сообщества, не для публичного сервиса с высокой нагрузкой.
- Только международная (латинская) азбука Морзе — кириллической нет.
- `TelegraphKey` калибрует точку/тире относительно выбранного `wpm`, а не
  адаптируется к реальному темпу пользователя — на непривычной скорости
  декодирование может быть неточным.
- `database/seed_callsigns.php` — раньше был публично доступен без защиты;
  теперь папка `database/` закрыта `.htaccess`, скрипт запускается только
  через консоль (`php database/seed_callsigns.php 200`).

## 🤖 Сжатая выжимка некоторых решений

Если тебя попросили доработать этот проект в новой сессии — вот сжатая
выжимка решений, которые уже приняты, чтобы не переспрашивать:

- **Стек закреплён**: PHP (без фреймворка) + MySQL через PDO + ванильный JS.
  Bootstrap сознательно не используется — своя дизайн-система в
  `assets/css/style.css` (переменные `--bg`, `--accent`, `--signal` и т.д.).
- **Аккаунтов нет и пока не планируется** — весь прогресс в
  `localStorage` через модуль `Progress` (`assets/js/progress.js`). MySQL —
  только для справочных данных (`achievements`, `callsigns`) и анонимной
  общей статистики (`global_stats`).
- **Схема БД меняется через миграции**: не редактируй `schema.sql` молча —
  если меняешь структуру/данные таблиц, добавляй новый
  `database/migration_N.sql` (следующий номер по порядку) и обновляй
  `schema.sql`, чтобы свежие установки и старые обновлялись согласованно.
  Обязательно обнови и раздел «История изменений», и список миграций в
  «Быстром старте»/«Деплое».
- **Сигнальная линия и лампа — сквозной элемент дизайна**, используется на
  каждой странице тренировки (`SignalLine`, `MorseLamp` из `signal.js`).
  Новые режимы тренировки должны их подключать для консистентности.
- **wpm-контролы — везде range-слайдеры с шагом 1**, не `<select>` с
  пресетами — это осознанное решение пользователя, не откатывай на select.
- **XP не должен фармиться** повторением одного и того же тривиального
  действия — при добавлении новых источников XP закладывай защиту по
  аналогии с `learn.js` (XP только пока символ/условие не «закрыто»).
- **XP всегда целые числа** — `Progress.addXp()` округляет через
  `Math.round`. Никогда не начисляй и не накапливай дробный XP напрямую —
  повторное сложение дробных чисел в JS даёт ошибку плавающей точки
  (`592.5000000000002`). Если формула даёт дробный множитель — округляй
  результат ДО вызова `addXp`, а не после.
- **Один `AudioContext` на весь сайт.** Никогда не создавай
  `new AudioContext()` напрямую в новом коде — используй
  `getSharedAudioContext()` из `audio.js`. У браузеров есть лимит на число
  одновременно живых контекстов; при создании нового на каждое
  воспроизведение (как было раньше) он рано или поздно исчерпывается,
  падает с ошибкой внутри `await`, и флаг "занято" (`isPlaying`/`recBusy`)
  зависает в `true` навсегда — тренировка молча замирает без звука и без
  реакции на клики. Все новые `await audio.play(...)` оборачивай в
  `try/catch/finally`, сбрасывающий флаг занятости в `finally`.
- Пользователь развёртывает локально через **Docker** (`docker compose up`,
  Windows 11 + Docker Desktop) — Laragon больше не используется. Деплоит на
  VPS (FastPanel, PHP 8.4) через `git pull`. CLI-скрипты (`seed_callsigns.php`
  и подобные) должны работать **без стрелочных функций (`fn`) и без строгой
  типизации параметров** — на некоторых хостингах `php` в консоли (SSH)
  указывает на более старую версию PHP, чем PHP-FPM самого сайта.
- При добавлении новой миграции — не забудь примонтировать её в
  `docker-compose.yml` (`db.volumes`), иначе на свежих Docker-установках
  она не накатится автоматически.
