## Что это

MorseWave — веб-тренажёр для изучения азбуки Морзе с нуля до приёма на слух
реальных позывных. Без фреймворков на фронтенде и без Bootstrap: чистый JS
(Web Audio API) + собственная тёмная дизайн-система.

Прогресс (XP, уровень, серия дней, выученные символы) хранится локально
в браузере — регистрация не нужна. MySQL используется под капотом для
справочных данных (достижения, банк позывных) и анонимной статистики
сообщества.

## Возможности

| Раздел | Что внутри |
|---|---|
| 📖 **Буквы** | Учи каждый символ отдельно: отправка ключом (тап/пробел, с живым сайдтоном) и приём на слух (играет символ → нажми на экранной клавиатуре) |
| 🎯 **Метод Коха** | Символы сразу на целевой скорости; новый символ открывается при точности ≥90%; можно вручную установить любое число открытых символов (в обе стороны) |
| 🔢 **Группы символов** | Приём на слух групп по 2–5 символов: буквы / цифры / микс / только выученные, скорость и интервалы Фарнсворта — слайдерами |
| 📡 **Позывные** | Тренировка на реальных по формату позывных (включая российские форматы), банк расширяется скриптом-генератором |
| 🏆 **Достижения** | 17 ачивок, условия хранятся в БД — можно добавлять новые без правки кода |
| 🎯 **Задание дня** | Единый набор параметров тренировки на сегодня для всех + бонус XP |
| 💡 **Сигнальная лампа** | Визуальный аналог звука — можно тренироваться совсем без звука |
| ⚠️ **Сброс прогресса** | Кнопка на странице «Достижения» — полный сброс localStorage с подтверждением |

## Стек

- **PHP 8+** — без фреймворка, простой роутинг по файлам
- **MySQL / MariaDB** — через PDO
- **Vanilla JS** — Web Audio API для звука, без сборки и npm
- **Кастомный CSS** — дизайн-система в `assets/css/style.css`, без Bootstrap

## Быстрый старт (Laragon, Windows)

```bash
cd C:\laragon\www
git clone <URL_ТВОЕГО_РЕПО> morse-trainer
```

1. Запусти Laragon (Apache/Nginx + MySQL) — хост `morse-trainer.test`
   поднимется автоматически.
2. Скопируй `config/database.example.php` → `config/database.php`
   (сам `config/database.php` не хранится в git — там могут быть боевые
   креды, поэтому он в `.gitignore`).
3. В HeidiSQL (кнопка **Database** в Laragon) или phpMyAdmin выполни
   **по порядку**: `database/schema.sql`, затем все `database/migration_*.sql`
   по возрастанию номера (см. список миграций ниже).
4. Проверь `config/database.php` (для Laragon по умолчанию подходит
   `host=localhost`, `user=root`, пустой пароль).
5. Открой `http://morse-trainer.test`.

Опционально — сгенерировать побольше позывных. Папка `database/` закрыта
`.htaccess` (защита от прямого доступа к SQL-файлам), поэтому запускать
скрипт нужно через консоль (в Laragon — кнопка "Terminal"/Cmder):
```bash
php database/seed_callsigns.php 200
```

## Как это устроено внутри

- **`assets/js/audio.js`** — `MorseAudio` проигрывает точки/тире через
  осциллятор Web Audio API, поддерживает интервалы Фарнсворта (символы на
  одной скорости, паузы — на другой, более медленной). `onSymbol` колбэк
  отдаёт `durationMs` — используется для точной синхронизации визуализации.
- **`assets/js/input.js`** — `TelegraphKey` превращает удержание клавиши
  (тап по экрану или пробел) в точки/тире по длительности нажатия
  относительно текущего `wpm`, плюс живой сайдтон (реальный звук) во время
  удержания. Пороги: `< 2×unit` → точка, иначе тире; пауза `2.5×unit` —
  граница буквы, ещё `+4×unit` — граница слова.
- **`assets/js/signal.js`** — `SignalLine` (осциллограф из точек/тире) и
  `MorseLamp` (визуальная лампа) — сквозные визуальные элементы, синхронно
  отражающие и звук, и ввод на всех страницах тренировки.
- **`assets/js/progress.js`** — весь прогресс пользователя в одном ключе
  `localStorage` (`morsewave_progress_v1`): XP, уровень, серия дней,
  выученные символы, уровень Коха, статистика, разблокированные
  достижения. `Progress.resetAll()` — полный сброс.
- **Достижения** определяются в MySQL (`achievements`, поле `condition_type`
  + `condition_value`), а проверка условий против статистики происходит на
  клиенте в `Progress.checkAchievements()` — сопоставление типов см. в
  `progress.js` (`statValue()`).
- **XP анти-фарм**: в `learn.js` XP за отправку символа начисляется только
  пока символ ещё не в `learnedLetters`; после освоения повторы бесплатны.

## Деплой на VPS

```bash
cd /var/www
git clone <URL_ТВОЕГО_РЕПО> morse-trainer
cd morse-trainer
cp config/database.example.php config/database.php
mysql -u root -p < database/schema.sql
mysql -u root -p morse_trainer < database/migration_2.sql
mysql -u root -p morse_trainer < database/migration_3.sql
mysql -u root -p morse_trainer < database/migration_4.sql
mysql -u root -p morse_trainer < database/migration_5.sql
mysql -u root -p morse_trainer < database/migration_6.sql
```

Дальше:

1. Виртуальный хост Apache/Nginx на папку проекта (document root — корень
   репозитория, где лежит `index.php`).
2. Убедиться, что установлены `php`, `pdo_mysql`.
3. Обновить `config/database.php` под продакшн-креды MySQL (для публичного
   сервера лучше вынести их в переменные окружения).
4. Обновления — `git pull` на сервере после `git push` с локальной машины,
   плюс накатить новые `migration_*.sql`, если они появились.
5. **Apache**: в конфиге виртуального хоста должно быть `AllowOverride All`
   (или хотя бы `AllowOverride AuthConfig Limit`) для директорий `config/`
   и `database/` — иначе `.htaccess`-запреты на эти папки не сработают.
   Большинство панелей управления (FastPanel, ISPmanager, cPanel) включают
   это по умолчанию для пользовательских сайтов.
   **Nginx** `.htaccess` не читает вообще — если сервер на Nginx, запрет
   нужно продублировать в конфиге сайта:
   ```nginx
   location ~ ^/(config|database)/ { deny all; return 404; }
   ```

## История изменений

  символов 2–5, позывные, XP/уровни/серия дней, 15 ачивок. Прогресс в
  localStorage, справочные данные — в MySQL.

    (`migration_2.sql`).
  - Сайдтон при отправке ключом; звук символа при верном повторе.
  - Порог «выучено» 3 → 5 повторов; фикс фарма XP на выученных символах.
  - Визуальная лампа (`MorseLamp`) во всех тренажёрах; сигнальная линия и
    лампа сбрасываются перед каждым проигрыванием.
  - Скорость — слайдеры с шагом 1 везде; Фарнсворт — чекбокс + слайдер.
  - Ввод ответа в Кохе/группах/позывных доступен сразу, не дожидаясь конца

  - Ручная установка уровня Коха; российские форматы позывных +

  - README/LICENSE под GitHub, `config/database.php` вынесен из git.

  - Ачивка «Мастер Коха» требовала 40 символов при 38 существующих —

  - Слайдер ручной установки уровня Коха разрешал двигать только вперёд —

  - **XP-фарм в «Буквах» (отправка ключом)**: раньше опыт начислялся за

    переписан рендер списка с защитой от отсутствующих полей.

  - В `morse-data.js` был случайно оставлен **дублирующий черновик**
    `ABBREVIATIONS` (с полем `desc` вместо `meaning`) — грузился раньше
    актуального `abbreviations.js` и побеждал, а второе объявление того же
    имени переменной падало с `SyntaxError: Identifier 'ABBREVIATIONS' has
    already been declared`. Отсюда и код показывался верно (поле `code`
    было в обоих), и описание всегда было «без описания» (поле называлось
    `desc`, а не `meaning`). Дубль удалён из `morse-data.js`.
  - На всякий случай `ABBREVIATIONS` объявлен через `var`, а не `const` —
    если что-то похожее случайно продублируется снова, будет тихая
    перезапись вместо падения всей страницы.
  - Все CSS/JS-файлы теперь подключаются с `?v=<время изменения файла>`
    (`asset_v()` в `includes/header.php`) — браузер больше не должен
    показывать старую закэшированную версию после обновления файлов.
    Если баг всё же похож на кэш — теперь достаточно обычного обновления
    страницы, жёсткая перезагрузка (Ctrl+F5) не нужна.
- **v2.9 — защита перед публичным деплоем.**
  - Добавлены `config/.htaccess` и `database/.htaccess` — запрещают прямой

    больше нельзя открыть по прямой ссылке из браузера на проде).
  - `database/seed_callsigns.php` теперь запускается только через CLI —

    `.htaccess` блокирует папку `database/` везде, где Apache его читает,
    включая Laragon по умолчанию — это осознанный компромисс ради
    безопасности на проде.
- **v2.10 — фикс пустого condition_type (`migration_5.sql`).**
  - У ачивок `streak50`/`streak500` при накатке миграций по отдельности

    Для новых установок через актуальный `schema.sql` этот баг не
    воспроизводится — там всё в одном файле и в правильном порядке.

    учитывает реально прозвучавшие группы в общем счётчике `groups_50/500`,

## Известные ограничения / открытые вопросы

- Нет аккаунтов — прогресс привязан к браузеру/устройству, при чистке
  localStorage или смене браузера обнуляется.
- Только международная (латинская) азбука Морзе — кириллической нет.
- `TelegraphKey` калибрует точку/тире относительно выбранного `wpm`, а не
  адаптируется к реальному темпу пользователя — на непривычной скорости
  декодирование может быть неточным.
- `database/seed_callsigns.php` — раньше был публично доступен без защиты;
  теперь папка `database/` закрыта `.htaccess`, скрипт запускается только
  через консоль (`php database/seed_callsigns.php 200`).

## 🤖 Сжатая выжимка некоторых решений

Если тебя попросили доработать этот проект в новой сессии — вот сжатая
выжимка решений, которые уже приняты, чтобы не переспрашивать:

- **Стек закреплён**: PHP (без фреймворка) + MySQL через PDO + ванильный JS.
  Bootstrap сознательно не используется — своя дизайн-система в
  `assets/css/style.css` (переменные `--bg`, `--accent`, `--signal` и т.д.).
- **Аккаунтов нет и пока не планируется** — весь прогресс в
  `localStorage` через модуль `Progress` (`assets/js/progress.js`). MySQL —
  только для справочных данных (`achievements`, `callsigns`) и анонимной
  общей статистики (`global_stats`).
- **Схема БД меняется через миграции**: не редактируй `schema.sql` молча —
  если меняешь структуру/данные таблиц, добавляй новый
  `database/migration_N.sql` (следующий номер по порядку) и обновляй
  `schema.sql`, чтобы свежие установки и старые обновлялись согласованно.
  Обязательно обнови и раздел «История изменений», и список миграций в
  «Быстром старте»/«Деплое».
- **Сигнальная линия и лампа — сквозной элемент дизайна**, используется на
  каждой странице тренировки (`SignalLine`, `MorseLamp` из `signal.js`).
  Новые режимы тренировки должны их подключать для консистентности.
- **wpm-контролы — везде range-слайдеры с шагом 1**, не `<select>` с
  пресетами — это осознанное решение пользователя, не откатывай на select.
- **XP не должен фармиться** повторением одного и того же тривиального
  действия — при добавлении новых источников XP закладывай защиту по
  аналогии с `learn.js` (XP только пока символ/условие не «закрыто»).
- Пользователь разворачивает всё через **Laragon на Windows 11**, деплоит на
  VPS через `git pull`. Правки должны оставаться совместимыми с этим
  процессом (простые `.sql`-файлы, никаких миграций, требующих CLI-фреймворков
  вроде Artisan/Doctrine).
